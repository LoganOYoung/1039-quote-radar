# 门户页面规划

面向「1039报价雷达」的首页/门户页规划：**优先国内手机微信打开**，区块结构、内容要点、流程提示。

---

## 〇、国内手机微信打开的使用习惯（最高优先级）

门户页与所有关键页（首页、生成报价、报价查看）**必须按「微信内打开 + 竖屏手机」优先**设计。

### 0.1 微信内置浏览器与环境

| 项 | 说明与建议 |
|----|------------|
| **内核** | 微信内为 X5/Chromium，支持现代 CSS；需测 iOS 微信与安卓微信表现一致 |
| **分享** | 链接在聊天里打开多为「微信内浏览器」，标题与描述来自 OG/页面 title，需设好 `title`、`description` 和分享图（若有） |
| **字体** | 微信可能放大系统字体，正文建议 ≥ 15px（约 `text-base`），避免小字被放大后仍难读或布局错位 |
| **横屏** | 多数用户竖屏使用，布局以竖屏单列为主，不依赖宽屏 |

### 0.2 触控与点击区域

| 项 | 建议 |
|----|------|
| **可点区域** | 按钮、链接等最小点击高度 **≥ 44px**（约 `min-h-[44px]` 或 `py-3`），避免误触或难点 |
| **间距** | 主 CTA 与次 CTA 之间留足间距（如 12～16px），防止误点 |
| **主操作** | 「生成报价链接」放首屏偏下或中部，拇指易触；不依赖 hover，用 `:active` 或 tap 反馈即可 |

### 0.3 首屏与滚动

| 项 | 建议 |
|----|------|
| **首屏内容** | 品牌名 + 一句价值/痛点 + **主 CTA 按钮** 尽量一屏内完成；亮点列表可 2～3 条精简或收起，其余「展开/更多」 |
| **少横滑** | 不做必须横滑的轮播；若多步骤用竖排或分页点击 |
| **滚动** | 内容尽量短，减少长滚动；若必须长页，底部再次出现主 CTA |

### 0.4 安全区与全屏

| 项 | 建议 |
|----|------|
| **刘海 / 挖孔** | 顶部留出 `env(safe-area-inset-top)`，避免标题被遮挡 |
| **底部横条** | 底部主 CTA 或固定操作栏留出 `env(safe-area-inset-bottom)`，避免被 Home 条挡住 |
| **viewport** | 使用 `viewport-fit=cover` 以便安全区生效（如 `<meta name="viewport" content="width=device-width, viewport-fit=cover">`） |

### 0.5 性能与加载

| 项 | 建议 |
|----|------|
| **首屏速度** | 首屏不依赖大图或外链字体，减少微信内打开白屏时间 |
| **弱网** | 文案先出、图片可懒加载；主 CTA 尽早可点 |

### 0.6 与规划的关系

- **区块规划**：Hero 以「标题 + 主 CTA」为主，亮点可收短或折叠。
- **顶栏**：移动端顶栏可简化（仅 Logo + 主 CTA），或与 Hero 合并，减少首屏高度。
- **流程 4 步**：移动端可收为「展开查看」或单列 4 行短句，避免首屏过长。
- **页脚**：链接做足点击区域，版权小字放最底。

## 一、门户页定位

| 维度 | 说明 |
|------|------|
| **主要访客** | 义乌/外贸商户、业务员（销售）、小外贸公司 |
| **来访目的** | 了解工具能做什么、是否可信、如何开始用 |
| **核心动作** | 点击「生成报价链接」进入 `/quote/new`，或登录/进仪表盘 |
| **次要动作** | 进仪表盘看历史报价与访问数据、进管理后台（若有权限） |

门户页需在 **3～5 秒内** 让人明白：这是给外贸报价用的工具，能保护价格、锁定汇率、看到客户打开动态。  
**移动端**：首屏完成「看懂 + 点主 CTA」，少滚动、大按钮。

## 二、区块规划（自上而下）

### 1. 顶部导航（可选）

| 项 | 建议 |
|----|------|
| 左侧 | Logo + 产品名「1039报价雷达」 |
| 右侧 | 「生成报价」→ `/quote/new`；「仪表盘」→ `/dashboard`；（可选）「说明/帮助」→ 文档或锚点 |
| 风格 | 简洁、与当前深色+翠绿一致，不抢主内容 |
| **移动/微信** | 顶栏可收窄或与 Hero 合并；右侧链接做 **≥44px 高** 可点区域；留出顶部安全区 |

---

### 2. Hero（首屏）

| 元素 | 建议 |
|------|------|
| **主标题** | 一句价值主张，如：「报价链接，可防可追踪」或「让报价单会说话」 |
| **副标题/一句说明** | 如：客户点开链接，谁看了、看了多久，你都知道；价格可锁、可申请解锁。 |
| **主 CTA** | 按钮「生成报价链接」→ `/quote/new`，主色高亮；**移动端 ≥44px 高、可全宽**，拇指易点 |
| **次 CTA** | 按钮「进入仪表盘」→ `/dashboard`，描边或次要样式；与主 CTA 留足间距防误触 |
| **视觉** | 保留现有雷达图标 + 深色背景，可加一句「免费使用」增强信任 |
| **移动/微信** | 主 CTA 尽量首屏内可见、无需滚动；正文字号 ≥15px；顶部留安全区 |

当前首页已具备：Logo+名、痛点句、三个亮点、双按钮。规划时在现有基础上**优先保证移动端首屏一屏内完成「看懂+点主 CTA」**。

---

### 3. 功能亮点（当前已有，可略扩展）

保持 3 个核心点，每点 **图标 + 标题 + 一两句说明**：

| 亮点 | 标题建议 | 说明要点 |
|------|----------|----------|
| **价格防护** | 价格防护锁 | 客户点申请，你点同意，价格才显示；防截图转发给同行 |
| **汇率锁定** | 汇率平衡器 | 报价页展示锚定汇率，结汇不因短期波动吃亏 |
| **访问可见** | 同行防火墙 / 访问雷达 | 谁打开、看了多久、多 IP 异常可提醒；保护商业机密 |

可选：增加第 4 点「**报价计算**」——FOB/CFR/CIF、散货整柜、附加费，一页算清（链到 `/quote/new` 的到岸价区块）。

---

### 4. 使用流程（可选新区块）

用 3～4 步说明「从零到发出报价链接」：

| 步骤 | 标题 | 说明 |
|------|------|------|
| 1 | 填报价 / 智能粘贴 | 产品名、EXW、利润率等；可粘贴聊天记录自动识别 |
| 2 | 生成链接 | 一键生成报价链接，可选锁定汇率、受控访问 |
| 3 | 发给客户 | 微信/邮件发链接，客户打开即被记录 |
| 4 | 看仪表盘 | 查看谁看了、停留时长、解锁申请，及时跟进 |

可配简单示意图或图标，减少认知负担。

---

### 5. 信任与说明（可选）

| 内容 | 建议 |
|------|------|
| **免费** | 明确写「免费使用」，无隐藏收费 |
| **数据与隐私** | 简短说明：数据存于云端、仅您本人可见（仪表盘），不对外展示 |
| **适用对象** | 一句话：面向 1039 市场采购、外贸公司、出口商户的报价与跟进工具 |

若后续有「帮助中心」或文档站，可在此放链接。

---

### 6. 主 CTA 再次强调（可选）

在页面中下再来一次双按钮（生成报价 + 仪表盘），方便长页滚动后仍能快速行动。

---

### 7. 页脚（可选）

| 项 | 建议 |
|----|------|
| 链接 | 首页、生成报价、仪表盘；若有隐私政策/使用条款可放此 |
| 版权 | 「1039报价雷达」+ 年份 |
| 风格 | 弱化视觉，小字、低对比 |

## 六、实施优先级建议

| 优先级 | 内容 | 说明 |
|--------|------|------|
| **P0** | **移动/微信体验** | 大按钮(≥44px)、首屏主 CTA、安全区、正文≥15px；先于桌面版优化 |
| P1 | Hero + 双 CTA + 三个亮点 | 已有，按移动端收短亮点或折叠，文案微调 |
| P2 | 顶部导航 | 各子页可共用；移动端可简化或与 Hero 合并 |
| P3 | 使用流程 4 步 | 降低首次使用门槛；移动端可收起或单列短句 |
| P4 | 信任与说明 + 页脚 | 增强信任与合规感；页脚链接做大点击区 |

按需选择实施；**国内手机微信打开优先**，再考虑桌面与长页扩展。
